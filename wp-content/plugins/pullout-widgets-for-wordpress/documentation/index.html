Plugin for WordPress

Turn regular widgets into slick sliding-out pullouts in seconds!

Quick Start Video Guide

Installation

  1. After you downloaded the plugin to your hard drive, login into your WordPress Dashobard and navigate to Plugins => Add New.
  2. Click Upload at the very top. Choose the Zip package file and click Install Now.
  3. After you receive confirmation "Plugin installed successfully", click Activate Pluign right below it. You can also Activate/Deactivate this plugin by navigating to Plugins => Installed Plugins.

Creating PullOuts

Creating PullOuts is as easy as adding widgets to your sidebar. Pullout options will be added at the bottom of each active widget.

So, go to the Appearance => Widgets section. After draggin a desired widget into PullOut Widgets Container sidebar expand it. At the bottom of the widget check "Turn Pullout On" box. Then click on Pullout Options button and options dialog will pop-up.  Set the preferences you require and then click Save. That's it. If you refresh your page with the widget, it should be displayed as a pullout.

Pullout Widget Options

Positioning

Screen Side

The side of the screen where the widget will be positioned (top, right, bottom, left).

Position on the Side

Position/offset of the pullout widget along the side of the screen. Can be set in pixels or percentages. Calculations are done from left to right on top and bottom sides ( where very left is 0% and very right is 100%) and top to bottom on left and right sides (where top is 0% and bottom is 100%).

Scroll with Page

By default this option is turned off and therefore the positioning of the widget on the side is relative to the dimensions of the windows. When this option is turned on, the porisioning of the widget on the side is retalive to the dimensions of the content rather than the window. Therefore widget is percieved as scrolling together with the content.

Behavior

Show/Slide Out

On Click - a widget is revealed when mouse clicks on its tab.

On Mouse Over - a widget is revealed when mouse hovers over its tab.

Once - when Element Appears on Screen [VIDEO DEMO #1] - this option is useful for auto-revealing widgets when specified elements are below the fold. For example, by specifying an element with ID #footer, our subscription widget will reveal iself automatically when user scrolls down to the end of the page. This action will happen only "once" every time the page is loaded. So, if user reaches footer multiple times on the same page - the widget will only reveal itself once automatically. But it will function this way every time the page is re-loaded.

Always - when Element Appears on Screen [VIDEO DEMO #2] - the same as option "once" (above), with the exception that widget will automatically reveal itself every time a specified element appears on the screen.

Once - After X Seconds* [VIDEO DEMO #2] - a widget will automatically reveal itself after specified number of seconds from the moment the first page with the current widget is loaded. If timer is set for a longer time frame, the trigger should work throughout multiple pages as well. For example: if you set a widget to automatically reveal itself after 0.5 seconds, it will show up almost immediately after the page is loaded. The option "once" assures that it happens only once on the site. If you set the timer to 180 seconds, then the widget will reveal itself after 3 minutes. The countdown of the time continues even if user moves to the next page of your site. As long as the current widget exists on other pages, it will get triggered automatically after the specified time frame.

Always - After X Seconds [VIDEO DEMO #2]  - same behavior as "Once - After X Seconds" with the exception that the timer will start countdown every time a user visits your website.

*Once - After X Seconds behaviour is using 30-day cookie to determine if a widget needs to be triggered or not. After the cookie expires, the behavior will be triggered again for returning users.

Once - After Visiting X Pages** [VIDEO DEMO #3] - a widget will automatically reveal itself after user visits the specified number of pages on the site. The count begins from the page where the widget is loaded for the first time. The widget will get triggered only on the pages where it rendered.

Always - After Visiting X Pages [VIDEO DEMO #3] - same behavior as "Once - After Visiting X Pages" with the exception that the counter will be reset after X pages and widgets will appear every time the user views X number of pages. For example, if the option is set to 3 - the widget will automatically reveal itself on every 3-rd page. So, if a user views 7 pages - widgets will show up twice during the user's stay on your site.

**Once - After Visiting X Pages behavior is using 1-hour cookie, which means that widgets with this option will get triggered only one time within an hour of browsing of the site by the same user.

Easing Effeects - these are animated sliding effects that you can apply to your pullout widgets. For more examples on the effects check ot this demostration.

Sliding Speed - sliding animation speed of the widget in miliseconds. The desired speed can very greatly depending on the selected Easing Effect. For example - "linear" easing effect works well with a speed of 500 miliseconds, while "easeOutBounce" will seem too fast and unattractive with the same speed, and rather works much better with 1500 miliseconds.

 

Style

Widget Width

A width of the widget. By default widgets will try to auto-adjust their width and appear nicely as is. But sometimes they may span across the whole screen or be not whide enough for your content. You can always re-djust the width manually via this option.

Background, Border, Text and Link Colors

By default style of your widget content is inhereted from your theme. If you place your pullout widgets into a different from the PullOut Widgets Container sidebar, then the widgets will inherit the styles of that particular sidebar. Current color options allow you to adjust the color to your desires.

Trim Styles

Trim styles offer you options to add a border to your widgets as well as rounded corners***. 

***Rounded Corners is a CSS3 options and only supported by modern browsers. IE6, IE7 and IE8 do not support this option.

 

PullOut Tab

Video Demo #4 for PullOut Tab Options

Tab Offset

Positioning of the tab along the side of the pullout widget.

Vertical Tab

By default all tab labels are rendered from left to right. Vertical orientation allows you to rotate the label vertically and is useful for longer labels. This is a CSS3 options and is not supported by IE6, IE7, IE8 and IE9.

Tab Icons

An image icon can be displayed in the tab of a pullout widget.

Open/Close Labels

You can enter custom label names for the opened and closed state of the pullout widget. For example, you can set the labels to Open Me and Close Me. When the widget is in a closed state, the label will display "Open Me". When the widget is opened state - the label will display "Close Me".

By default the tab label will use the Title of the widget. If the title of the widget is not set, then the widget will fall back on default "Open/Close" labels. You can always overwrite those labels in the current option.

If you wish to have a custom label different form the Title of the widget, it is sufficient to define only "Open" Label. In this case the Open Label and Close Label will be the same.

Hide Text Label

Tab will display selected icon only - no text label will be displayed.

Mobile Devices

Important: PullOut Widgets by default are turned OFF for mobile phones, but will work on tablets.

While PullOut Widgets plugin was not intended to be used on mobile devices, in my experience it works. Not always perfect, but it functions. On smaller devices, like phones, user experience is rather questionable - quite often, pullout widgets cover whole screen and it is hard to close them in smaller screen resolutions.

Since version 2.6 PullOut Widgets can be selectively turned off for phones, tablets or all mobile devices. By default, they are turned off for phones only.

Turn On/Off for selected device types

If you want to change the default option, add one of the definitions below to your wp-config.php file. I usually add it below define('WPLANG', ''); definition.

define('POW_OFF', 'phone'); //OFF on phones

Allow to display PullOut Widgets on all devices:

define('POW_OFF', false); //ON for ALL devices

OFF on Tablets:

define('POW_OFF', 'tablet'); //OFF on tablets
 

Most Frequent Issues

Issue 1:  When I click on Pullout Options link/button in the Widgets Dashboard - nothing happens.

Solution: 99% of the time this issues arises when a theme or a third party plugins loads jQuery UI library on the Widgets Dashboard page (widgets.php). Widgets Dashboard loads jQuery UI automatically and a duplicate library will case the Pullout Options to fail. To fix the issue, you need to identify which plugin or theme is loading the library:

Step 1: Click right mouse button anywhere on the widgets dashboard page and select View Source.

Step 2: In the source perform 2 searches: first for jquery-ui and second for jquery.ui. You might find 2 records of either of them or one of each. When reading the records, make sure they are references to the JavaScript and not CSS (JS tags will begin with <script and CSS - with <link rel="stylesheet"). Native WordPress library record should look like this:

<script type='text/javascript' src='http://mysite.com/wp-admin/load-scripts.php?c=1&amp;load=thickbox,admin-bar,jquery-ui-core,jquery-ui-datepicker,hoverIntent,common,jquery-color,jquery-ui-widget,jquery-ui-accordion,jquery-ui-mouse,jquery-ui-resizable,jquery-ui-draggable,jquery-ui-button,jquery-ui-position,jquery-ui-dialog,jquery-ui-sortable,jquery-ui-droppable,admin-widgets&amp;ver=fa7adaf29bfed546f3751df8d94c1ab1'></script>

WordPress load jquery-ui-core by default. Quite often a duplicate library is loaded by a custom theme from Google CDN and may look like this:

<script src="https://ajax.googleapis.com/ajax/libs/jqueryui/1.8.18/jquery-ui.min.js">

Step 3:  If you're fluent in PHP and WP coding - find the place where your theme or a plugin (if the library is loaded by a plugin, there might be another JS file loaded next to the library, which name you can identify by its URL) - and adjust it to load scripts wp_enqueue_script() function. Make sure the handle of jQuery UI library is 'jquery-ui-core' - this way WP will be able to determin if the library has already loaded or not and therefore avoid loading duplicates.

If Step 3 makes no sense to you, forward these instructions to your theme developer and ask him/her to fix your theme.

 

Issue 2: JavaScript Error in the Widgets Dashboard: 

Uncaught TypeError: Object [object Object] has no method ‘propAttr’.

Solution: See solution for Issue 1.

 

Issue 3 : When I expend widgets in the Widget Dashboard, Pullout Options are being displayed inside the widget and not in a pop-up dialog.

Solution: This happens when admin_widgets.js is not being loaded on the Widgets Dashboard page (widgets.php) - most often the folder's name contains spaces. 

Step 1: Click right mouse button anywhere on the Widgets Dashboard page and select to view source.

Step 2: In the source perform a search for admin_widgets.js. You should be able to find something like this:

<script type='text/javascript' src='http://mysite.com/wp-content/plugins/pullout-widgets/js/admin_widgets.js?ver=1.0'></script>

Step 3: Make sure that the URL displayed within the "src" attribute is correct. Make sure that the PullOut Widgets plugins' folder doesn't contain any weird characters - only letters, digits, periods, hyphens and underscores supported. Navigato to the folder via your FTP browser and re-name the folder if necessary.

 

Issue 4: Extra horizontal (or vertical) scrolling bar appears when PullOut Widgets plugin is active.

Solution: You will need to edit a /css/pullouts.css file that is located in the plugins' folder. Comment out the first "body" rule that appears at the top of the stylesheet. Simply wrap it with /* and */ like this:

/*
body {
    overflow-x: hidden;
    position: relative;
}
*/

 

Issue 5: PullOut Widgets appear in a sidebar instead of on the sides of the screen.

Solution: Make sure you place the PullOut Widgets inside of the PullOut Widgets Container on the Widgets Dashboard page. If this doesn't help - see solution for Issue 6.

 

Issue 6: I placed the widgets into the Widgets Pullout Container, but they don't appear on any of my pages.

Solution: Look around the page and see if any of the pullout widgets are rendered anywayere on the page. Check the bottom of the page as well, since this is where the PullOut Widgets Container is supposed to be loaded. If nothing found, click right mouse button on the page and select view source - search for any part of the content within the pullout widgets. If no sign of pullout widgets within the source code, that means that your theme doesn't use the wp_footer() function in its templates. What you need to do, is to edit your page.php template and add the following at the very bottom:

<?php wp_footer(); ?>

You might need to repeat the same for other templates as well: home.php, front-page.php, single.php and archive.php. If you have difficulties, please contact your theme developer and point him/her to this solution.

 

Issue 7: Pullout Widgets load but don't function on the page.

Solution: Most of the time the PullOut Widgets fail to function when a custom theme is loading an old jQuery library. WordPress comes with jQuery by default, but some theme authors ignore (or unaware of) this fact and load the library from Google CDN. Perform the following steps:

Step 1: Click right mouse button anywhere on the page and select View Source.

Step 2: Within source perform a search for jqueryi and jquery-ui. You might find the following:

1. You might find that there are multiple jQuery libraries are loaded.

2. WP comes with jQuery version 1.7.1, some themes may load older version of this library:

<script src="https://ajax.googleapis.com/ajax/libs/jquery/1.5.1/jquery.min.js">

If any of the above is true, then proceed with Step 3.

Step 3: if you're familiar with PHP coding, find the place in your theme where the jQuery library is loaded and simply comment that part out. Then add the following line below the one you commented out:

wp_enqueue_script('jquery');

Also, look around the same code and comment out the following function (if you find it) like this:

//wp_deregister_script('jquery');

If none of the above issues/solutions match your case, feel free to email me directly at max.chirkov@gmail.com

Other Notes

PullOut Widgets plugin comes with 2 sets of icons:

  • default  - /images/icons.png (dark purple color);
  • optional - /imags/icons-gray.png (gray color);

You can easily change icon color to gray by editing /css/pullouts.css and changing background file name in the following rule: ".pullout-button .icon".
